          How it works
          Link + receipt
        

        

          

            

              
    
                
  1. You write your message normally (email/notice) and plan to send it yourself.
    2. 
                
  2. This page hashes recipient + subject + body locally (SHA-256).
    3. 
                
  3. ACKZERO returns a Proof ID + claim link + receipt link (JSON).
    4. 
                
  4. You include the claim link in your message and send it.
    5. 
                
  5. Anyone can verify later via the signed receipt JSON.
    6. 
              

              

                ACKZERO proves a verifiable receipt existed at a time, and can record that the claim link was opened.
                It does not prove identity or guaranteed delivery.

          FAQ
          Clear limits
        

        

          

            Does ACKZERO store my message?
            
No. Hashing happens locally in your browser. ACKZERO stores hashes + timestamps only.

          

          

            Does this prove the recipient read the message?
            
No. It can record that the claim link was opened (availability). Reading is not guaranteed.

          

          

            Is this equivalent to registered electronic delivery?
            
No. ACKZERO is a technical receipt. Legal equivalence depends on jurisdiction and regulated providers.

          

          

            What should I include in my email?
            
Include the claim link. Optionally include the receipt link for transparency.

          

          

            Can anyone verify later?
            
Yes. The receipt link returns a signed JSON record that can be checked independently.
