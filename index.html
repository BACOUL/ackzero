API (v0.1)

v0.1 is ‚Äúlink + tracking‚Äù (no SMTP). You create a proof, embed the claim link in your message, and verify publicly later. The service stores hashes and timestamps only.

POST /v1/proofs

Creates a proof in KV and returns claim, receipt, and optional pixel URLs.

GET /p/:token

Public claim page. Logs an access event (availability proof).

GET /v/:id

Public JSON receipt signed with HMAC over a canonical string.

Example request
Content-Type: application/json
POST https://ackzero.example.workers.dev/v1/proofs
{
  "to": "recipient@example.com",
  "subject": "Cancellation notice",
  "body": "Hello, I hereby cancel...",
  "meta": { "channel": "email" }
}
cURL
Create a proof
curl -sS -X POST "https://ackzero.example.workers.dev/v1/proofs" \
  -H "content-type: application/json" \
  -d '{"to":"recipient@example.com","subject":"Cancellation notice","body":"Hello, I hereby cancel...","meta":{"channel":"email"}}'
Tip: for maximum privacy, your client can send hashes.to, hashes.subject, hashes.body instead of raw values.
