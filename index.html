What ACKZERO v0.1 does

v0.1 is intentionally simple: link + receipt. It does not send emails. It does not require accounts.

Issue a receipt

Create a proof record with POST /v1/proofs. Only the hash is required. The server signs the receipt and returns a claim link.

POST /v1/proofs HMAC signature

How it works

Flow v0.1: Hash locally ‚Üí Create receipt ‚Üí Insert claim link ‚Üí Events update ‚Üí Verify anytime

Minimal curl

curl -X POST "https://ackzero.io/v1/proofs" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"hash":"<sha256>","kind":"email","meta":{"subject":"Termination notice"}}'

# response includes:
# - id
# - token
# - urls.claim  => https://ackzero.io/p/<token>
# - urls.receipt => https://ackzero.io/v/<id>
# (optional) urls.pixel => https://ackzero.io/px/<id>.gif

Hash locally

const buf = new TextEncoder().encode(messageText);
const dig = await crypto.subtle.digest("SHA-256", buf);
const hash = [...new Uint8Array(dig)].map(b => b.toString(16).padStart(2, "0")).join("");

Create receipt

const r = await fetch("/v1/proofs", {
  method: "POST",
  headers: { "Content-Type":"application/json", "Accept":"application/json" },
  body: JSON.stringify({ hash, kind:"email", meta:{ subject:"Example" } })
});
const receipt = await r.json();
